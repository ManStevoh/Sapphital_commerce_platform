# Chapter 04: Block Library

**Document ID:** SCP-CMS-001-04  
**Version:** 1.0.0  
**Status:** 📝 Draft  
**Traceability:** ADR-003, Proposed ADR-013, FR-CMS-001, FR-CMS-012, NFR-039  

---

## Purpose

Define the platform-owned and theme-extendable **section and block catalog**, including rich-text (BlockNote) blocks for blog/lessons, commerce embeds, education embeds, and security constraints for embeds.

## Scope

- Core section types
- Nested block types
- BlockNote document blocks
- Commerce and education embeds
- Allowlisted oEmbed providers
- Accessibility and performance budgets per block

## Out of Scope

Theme-specific branded sections (Volume 6 Theme Store), plugin App Blocks registration API details (Volume 12).

## Two Block Systems

| System | Format | Used for |
|--------|--------|----------|
| **Theme sections/blocks** | ADR-003 JSON | Page builder layouts |
| **BlockNote document** | BlockNote JSON (proposed ADR-013) | Blog posts, lesson text, simple page bodies |

They share design tokens from SDS (Volume 4) but different schemas. Theme sections wrap commerce data; BlockNote embeds reference entities by ID.

## Proposed ADR-013 — BlockNote Canonical Rich Text

**Decision:** Store BlockNote JSON as canonical; maintain `body_html` cache for storefront/RSS.

**Rejected:** Raw Tiptap-only UI (more custom chrome), Lexical (maturity), unrestricted HTML paste (XSS).

**Sanitization:** On save, strip disallowed marks/nodes; HTML cache generated by trusted renderer only.

## BlockNote Document Blocks

Slash-menu catalog for blog/lessons:

| Block | P0/P1 | Security notes |
|-------|-------|----------------|
| paragraph, heading, lists | P0 | Standard |
| quote, code, divider | P0 | Code: escaped |
| table | P1 | |
| callout | P0 | |
| image | P0 | Must be media_id from library |
| video | P0 | Hosted media or allowlisted oEmbed |
| product_embed | P0 | `product_id` only |
| course_embed | P0 | `course_id` only |
| collection_embed | P1 | |
| cta | P0 | Internal link types preferred |

**Paste policy:** External HTML is converted through BlockNote paste rules; `<script>`, event handlers, and `javascript:` URLs removed.

## Commerce Embed Behavior

```text
product_embed → Storefront API product(id) → render card
If product draft/deleted → show placeholder in editor; hide on storefront
Prices always live from Catalog (FR-021 Money)
```

## Education Embed Behavior

```text
course_embed → course + product join → title, level, price, CTA
Respects enrollment status on storefront when customer authenticated
```

## oEmbed / External Media Allowlist

| Provider | Allowed | SSRF notes |
|----------|---------|------------|
| YouTube | Yes | Fixed hostnames only |
| Vimeo | Yes | Fixed hostnames |
| Loom | Yes | Fixed hostnames |
| Cloudflare Stream | Yes | Tenant media preferred |
| Arbitrary URL | **No** | Prevents SSRF |

Fetch flow for oEmbed metadata:

1. Validate URL host against allowlist
2. Resolve DNS; reject private/link-local/metadata IPs (SSRF)
3. HTTP GET only to allowlisted oEmbed endpoints with timeout ≤3s
4. Store sanitized metadata; never proxy arbitrary byte ranges to attacker

Full SSRF controls in Chapter 10.

## Accessibility Requirements (WCAG 2.2 AA)

- Images require `alt` (empty alt only when decorative flag set)
- FAQ accordion keyboard operable
- Video embeds need captions when merchant-hosted; third-party caption responsibility documented in UI
- Color contrast via SDS tokens — blocks must not introduce free-form hex that fails contrast without warning
- Focus order preserved when reordering blocks

## Performance Budgets

| Block class | Budget |
|-------------|--------|
| Hero image | ≤ 200 KB WebP (NFR-011) |
| Autoplay video | Disallowed on mobile by default |
| Product/course grids | Max 24 items client-render; rest paginate |
| Font loads | Theme-controlled; blocks cannot inject extra font URLs |

## Permissions

Installing custom theme sections requires theme publish rights. Merchants cannot upload arbitrary React. Plugin App Blocks require reviewed plugin (Vol 12).

## Extension Points

1. Theme package registers additional section types in `theme.schema.json`
2. Plugins register App Blocks into allowed theme slots
3. AI may suggest block insertions but cannot invent unregistered types

## Testing Strategy

- Snapshot tests per core section
- XSS fixtures for BlockNote paste
- oEmbed SSRF test suite (metadata IP, DNS rebinding simulation)
- Axe checks on rendered section HTML

## Acceptance Criteria

- [ ] All P0 BlockNote blocks insert via slash menu
- [ ] Product and course embeds resolve live prices
- [ ] Non-allowlisted oEmbed URL rejected
- [ ] Image block without alt fails SEO checklist (and warn on publish)
- [ ] Theme can override `hero` presentation without changing JSON settings shape
- [ ] Paste of `<img onerror=...>` does not persist executable markup
